# 技術的整合性ガイド - 数値基準と表記統一

## 📊 評価基準の統一

### 基準選択の方針
**現行ベースライン（16.79 tok/s）をメイン基準**として採用し、過去ベースライン（15.02 tok/s）を履歴比較として補助的に使用。

#### 理由
- 同一セッション・同一環境での公正な比較
- 実際のWEBテスト結果との整合性
- より信頼性の高い評価基準

### 改善率算出の統一
```
改善率（×） = 測定値 ÷ 現行ベースライン（16.79 tok/s）
```

## 📝 表記統一のパッチ

### 1. 改善率の凡例
```markdown
> 改善率（×）は、同一セッションで計測した**現行ベースライン（16.79 tok/s）**を分母として算出。過去の参照ベースライン（15.02 tok/s）は履歴比較として別掲。
```

### 2. NPUモードの凡例
```markdown
> NPU モードは係数ベースの予測値（現行ベースライン × 1.79x）。単発サンプルの 66.85 tok/s は算術換算の参考値であり、公式比較は 30.06 tok/s を用いる。
```

### 3. 品質スコアの定義
```markdown
> 品質スコア = 0.5以内を可とする運用ガード基準。PPL非計測時は長さ正規化自己BLEUと拒否率で代理評価。
```

### 4. メモリ効果の表現
```markdown
> KV段階量子化：対FP16基準で最大75%削減。IOBindingは追加でフットプリント15%改善。同時適用の総合効果はワークロード依存（実測レンジ: 70-80%削減）。
```
